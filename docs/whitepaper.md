## 摘要

本库为 STM8 微控制器提供了一套完整的嵌入式算法实现，核心特色是  
**零循环 CRC 计算**。在 8 位架构上，通过 SWAP、SRL/RRC 等指令的组合，  
实现了无需查找表、无循环分支的 CRC-16 计算，为 Flash 极度受限的  
场景提供了新选择。

---

## 零循环 CRC 原理

### 核心思想

传统 CRC 实现用逐位循环（8 次/字节）或查表法（256 字节表）。  
零循环实现通过**位交叉异或**将 8 位输入坍缩为 1 位奇偶值，再  
基于该值直接计算 16 位结果。

### STM8 实现路径

```
8位输入
  → SWAP + XOR（4位交叉）
  → SRL + XOR（2位交叉）
  → 奇偶检测（INC + AND #2）
  → 条件修正（分支或查表）
  → 16位结果组合
```

## STM8 架构特性利用

### SWAP 指令的独特价值

STM8 的 `SWAP A` 指令在 1 周期内交换累加器的高低 4 位，这是  
ARM 没有的指令。利用它可以在 1 周期内完成 4 位交叉，而 ARM  
需要 `EOR + LSR #4` 两条指令。

### SRL + RRC 的多位移位

STM8 没有桶形移位器，但可以通过 `SRL`（逻辑右移）和 `RRC`（带进位  
循环右移）的组合，实现跨字节的多位移位。

## 各模块详解

### 1. crc_modbus_noloop.s

零循环 Modbus CRC-16，多项式 0xA001。

关键代码：
```asm
UpdateModbusCRC:
  PUSHW X          ; 临时存储
  XOR   A,($1,SP)  ; 与 CRC 低字节异或
  SWAP  A          ; 4位交叉
  XOR   A,($1,SP)
  ; ... 奇偶检测与修正 ...
```

### 2. crc16_ccitt_noloop.s

零循环 CRC-16-CCITT，多项式 0x1021。

特点：使用 `AND #$F0` / `AND #$0F` 掩码分离高低半字节，  
这是 STM8 特有的优化技巧。

### 3. md5.s

完整 MD5 实现，支持三种输出格式：
- Mode 0：二进制（16字节）
- Mode 1：ASCII 十六进制（32字符）
- Mode 2：Base64（24字符）

### 4. base64.s

Base64 编解码，支持 URL 安全变体（`-` `_` 替代 `+` `/`）。

编码状态机用 RAM0/RAM1 存储剩余位，避免查表。

### 5. pid.s

IQ10 定点 PID 控制器，公式：
```
output = Kp*error + Ki*integral + Kd*(error - last_error)
```

所有系数和变量用 16 位定点表示，小数部分 10 位。

## 应用场景

- **智能电表**：Modbus 通信 + 温度采集（NTC）
- **工业网关**：多协议 CRC 校验 + PID 控制
- **物联网终端**：MD5 固件签名 + Base64 数据编码
- **Bootloader**：CRC 固件完整性验证

## 结论

STM8 作为经典的 8 位微控制器，在成本敏感的工业场景中仍有广泛应用。  
本库通过算法优化而非硬件升级，为 STM8 赋予了处理复杂任务的能力。

零循环 CRC 的实现证明：**算法的深度比硬件的速度更重要**。  
在资源受限的环境中，对每一行代码的深思熟虑，往往比换用更强大的  
CPU 更有价值。
