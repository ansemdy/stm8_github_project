## 一、核心哲学：尊重硬件，敬畏时序

### 1.1 从田野到硅片

我出生在潍坊农村，年轻时种地，后来写代码。两件事道理相通：

- **种地**：尊重土地，了解墒情，不浪费一粒种子
- **写代码**：尊重硬件，了解时序，不浪费一个时钟周期

那种耐心和精准，来自田野，不只是实验室。

### 1.2 确定性高于速度

在工业现场，**确定性比峰值速度更重要**。

- 查表法快，但Cache Miss引入不可预测延迟
- 硬件CRC快，但总线仲裁可能插入等待状态
- **零循环算法**：慢一些，但周期数死死锁定，这才是硬实时

> "几纳秒的抖动在实验室无关紧要，但在航空航天产线，累积起来就是PID失控。"

## 二、位交叉异或降维：CRC的本质

### 2.1 传统视角的问题

CRC被教成"串行多项式除法"，于是大家逐位循环：

```c
for (i=0; i<8; i++) {
    if (crc & 1) crc ^= 0xA001;
    crc >>= 1;
}
```

8次循环，80+周期，还破坏了流水线。

### 2.2 数学视角的转换

CRC-16本质上是 **GF(2)上的线性变换**：

```
输入：8位字节 + 16位当前CRC = 24位信息
输出：16位新CRC
```

既然是线性变换，就可以**并行计算**，不需要串行模拟。

### 2.3 信息论降维路径

8位输入要映射到16位输出，信息论上存在最短路径：

```
8位 → 4位：XOR LSR #4  （位0-3与位4-7交叉）
4位 → 2位：XOR LSR #2  （位0-1与位2-3交叉）
2位 → 1位：XOR LSR #1  （位0与位1交叉）
```

3次交叉异或，8位坍缩为1位**奇偶校验值**。这个值决定是否应用多项式修正。

**关键洞察**：3次是**理论下限**。2次只能到2位，4次冗余。3次刚刚好。

### 2.4 硬件特性的极致利用

**ARM桶形移位器**：LSL/LSR/ROR在EOR指令中**免费执行**，不消耗额外周期。

```asm
EOR R0, R0, R2, LSL #15   ; bit0 → CRC bit15（免费移位）
EOR R0, R0, R2, LSR #1    ; bit1-7 → CRC高字节（免费移位）
EOR R0, R0, R2, ROR #18   ; bit0-1 → CRC低字节高2位（免费移位）
```

4次EOR完成16位修正，每次移位都是"免费"的。这是**指令级并行度最大化**。

**STM8的SWAP指令**：1周期内交换高低4位，ARM没有这条指令。

```asm
SWAP A          ; 4位交叉，1周期完成
```

不同架构有不同"免费午餐"，关键是找到它。

## 三、零循环设计：消灭分支预测

### 3.1 分支的代价

条件分支（if-else）在流水线处理器上：
- 预测正确：0周期惩罚
- 预测失败：3-15周期惩罚（取决于流水线深度）
- **在实时系统中，预测失败是不可接受的**

### 3.2 用算术运算替代逻辑判断

**V-USB的启示**：Christian Starkjohann在8位AVR上，用`neg`+`andi`生成全1/全0掩码，替代if-else。

**ARM上的进化**：

```asm
ANDS R3, R3, #1      ; 提取奇偶位，设置Z标志
MUL  R3, R3, LR      ; 如果R3=1则得0x1C0，否则0
```

用**乘法实现条件赋值**，无分支，确定性时序。

**STM8上的进化**：

```asm
INC  A
AND  A, #2
JRNE crc8005_xor_1   ; 短分支，STM8流水线浅，惩罚小
```

不同架构，不同策略，但目标一致：**消灭或减少分支**。

### 3.3 IT块的条件执行

Cortex-M3/M4的IT块：

```asm
ANDS Rswap, Rswap, #1
IT   NE
MOVNE Rswap, #0x1C0   ; 条件执行，无分支
```

比MUL更优，但需要架构支持。

## 四、IQ定点数学：重新理解数值

### 4.1 为什么不用浮点

- 浮点单元（FPU）耗电、占面积
- 很多MCU没有FPU（Cortex-M0/M3）
- 浮点运算**非确定性**（NaN、Inf、舍入模式）

### 4.2 IQ格式的本质

不是"模拟浮点"，而是**重新理解数值范围**：

| IQ格式 | 整数位 | 小数位 | 范围 | 精度 |
|--------|--------|--------|------|------|
| IQ15 | 16位 | 15位 | ±65536 | 0.00003 |
| IQ24 | 7位 | 24位 | ±128 | 0.00000006 |
| IQ30 | 1位 | 30位 | ±2 | 0.0000000009 |

**选择IQ格式 = 选择数值范围和精度的平衡**。

### 4.3 15°折叠的直觉

atan2的IQ24实现中，为什么选择15°作为折叠点？

因为 **tan(15°) = 2 - √3 ≈ 0.2679**，有特殊数学性质：

```
√3 ≈ 1.732
2 - √3 ≈ 0.268
```

折叠公式：
```
如果 |x| > tan(15°):
    x' = (√3·x - 1) / (√3 + x)
```

这个变换将任意角度压缩到 [-15°, +15°] 区间，而15°内可以用**低阶有理函数**精确逼近。

**为什么是15°不是10°或20°？**
- 10°：折叠次数多，效率低
- 20°：逼近区间大，需要高阶多项式
- **15°：刚好平衡，3次折叠覆盖全象限**

### 4.4 有理函数逼近

在 [-15°, +15°] 内，arctan(x) 可以用：

```
arctan(x) ≈ (x + a·x³) / (2π + b·x² + c·x⁴)
```

系数通过**最小最大逼近**（Remez算法）优化，确保最大误差均匀分布。

**为什么不用泰勒级数？**
- 泰勒级数在边界收敛慢
- 有理函数在全局收敛快
- **工业级实现需要全局一致精度**

## 五、跨架构移植的思维框架

### 5.1 同一算法，不同表达

| 操作 | ARM (Cortex-M4) | STM8 | AVR (V-USB) |
|------|-----------------|------|-------------|
| 4位交叉 | `EOR R3, R2, R2, LSR #4` | `SWAP A` + `XOR` | `SWAP` + `EOR` |
| 条件赋值 | `MUL R3, R3, LR` | `INC`+`AND`+`JRNE` | `NEG`+`ANDI` |
| 多位移位 | 桶形移位器免费 | `SRL`+`RRC`组合 | `LSR`+`ROR` |
| 寄存器压力 | 16个寄存器 | 5个寄存器 | 32个寄存器 |

**核心算法不变，表达方式随架构调整**。

### 5.2 移植 checklist

1. **数据通路宽度**：8位？16位？32位？
2. **移位能力**：免费？单周期？多周期？
3. **乘法器**：有？无？单周期？多周期？
4. **分支惩罚**：流水线深度？预测机制？
5. **寄存器数量**：够用吗？需要栈变量吗？

## 六、给后来者的建议

### 6.1 不要迷信硬件

GHz级主频掩盖不了算法低效。在资源受限场景中，**算法的深度比硬件的速度更重要**。

### 6.2 回归数学本质

优化不是试错调参，而是**理解问题的数学结构**。CRC是线性变换，atan是有理函数，PID是差分方程——找到本质，优化自然来。

### 6.3 尊重前辈，但超越前辈

V-USB的`neg`+`andi`是AVR上的杰作，但在ARM上可以用`MUL`或`IT`块做得更好。**学习思路，不拘泥形式**。

### 6.4 代码是情书，写给硅片

每一行汇编都是对硬件的深刻理解，每一个时钟周期都是与硅片的对话。这种**工艺精神**，是嵌入式开发的灵魂。
